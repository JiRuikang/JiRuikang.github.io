Vue源码阅读一

发表于 2018-03-01 | 分类于 前端 | 阅读次数 次

主要阅读npm run dev和npm run build的两条主线，首先来看npm run dev的执行主线：
1、 package.json
“scripts”: {
“dev”: “rollup -w -c scripts/config.js –environment TARGET:web-full-dev”,
打包工具：rollup
npm run dev执行文件：config.js
知识点：
rollup关键配置：
• entry 源代码的入口
• moduleName 这个参数只有设置了umd的时候才需要设置
• plugins 顾名思义，这里是来放插件的
• format 设置打包的模式，比如umd，cjs等
• dest 输出
• banner 可以在输出文件的头部写入一段你想写的
• alias 别名
• env 环境变量，比如process.env等
rollup与其他工具的集成：
rollup-plugin-buble：es6转es5
rollup-plugin-alias：用别名缩短path
rollup-plugin-commonjs：commonjs转es5（应该用在其他插件转换你的模块之前 - 这是为了防止其他插件的改变破坏CommonJS的检测）
rollup-plugin-node-resolve：如何安装外部模块
rollup-plugin-replace：可以将动态设置的变量提取出来，在配置文件中设置
2、 scripts/config.js
if (process.env.TARGET) {
module.exports = genConfig(process.env.TARGET)
} else {
exports.getBuild = genConfig
exports.getAllBuilds = () => Object.keys(builds).map(genConfig)
}
npm run dev*执行函数：genConfig(…)
“dev”: “rollup -w -c scripts/config.js –environment TARGET:web-full-dev”,
等同于执行rollup –w –config (config变量)将entry-runtime-with-compiler.js打包。
知识点：
module.exports与exports.XXXX：
知乎来讲：require方只能看到module.exports对象，我们在编写模块时用到的exports对象实际上是对module.exports对象的引用。
Node.js实战来讲：典型的模块是一个包含exports对象属性定义的文件，用module.exports微调模块的创建，module.exports可以对外提供单个的函数或变量，不必局限于一个对象。一个文件中既有module.exports又有exports时，exports被忽略。
Object.assign(target, source1,…):
Object.assign方法用来将源对象（source）的所有可枚举属性，复制到目标对象（target）。它至少需要两个对象作为参数，第一个参数是目标对象，后面的参数都是源对象。只要有一个参数不是对象，就会抛出TypeError错误。
Object.defineProperty(obj,prop,descriptor):
直接在一个对象上定义一个新属性，或者修改一个对象的现有属性， 并返回这个对象。默认情况下，使用Object.defineProperty()添加的属性值是不可变的。
3、 src/platforms/web/entry-runtime-with-compiler.js
知识点：
expr1 && expr2:
如果expr1 能转换成false则返回expr1，否则返回expr2。
也就是说el=el && query(el)等价于：
if(el){el=query(el)}
document.querySelector():
element 是一个 element 对象（DOM 元素）。
selectors 是一个字符串，包含一个或是多个 CSS 选择器 ，多个则以逗号分隔。
str.charAt():
返回一个新的字符串，该字符串由位于指定偏移量处的单个UTF-16代码单元组成。
